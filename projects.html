1) Pi-hole Home DNS Deployment (Production Home Network)

Goal: Reduce ad/tracker traffic and gain DNS-level visibility while maintaining network reliability for all users.

Environment: Raspberry Pi, Pi-hole, home network router (DNS forwarding), multiple client devices

  • What I did:
    • Deployed Pi-hole on a Raspberry Pi to function as the primary DNS resolver for the network.
    • Configured curated blocklists to block tens of thousands of domains associated with advertising, telemetry, and known malicious activity.
    • Validated DNS resolution behavior to ensure normal browsing and streaming services were not disrupted.
    • Developed and used a small maintenance script to automate system updates, Pi-hole gravity updates, cache cleanup, and service restarts.
    • Monitored DNS query logs and rate-limiting behavior to identify noisy or misconfigured clients.
  • Security & operational considerations:
    • Avoided overblocking by gradually tuning blocklists and using targeted whitelisting when required.
    • Ensured DNS availability to prevent single points of failure for the network.
    • Maintained separation between configuration details and public documentation to avoid exposing sensitive information.
  • What I learned:
    • How DNS-based filtering works and where its strengths and limitations exist.
    • How excessive DNS queries can indicate background services, misconfigured devices, or privacy-invasive applications.
    • The importance of automation and routine maintenance for security infrastructure, even in small environments.

2) Incident Response Log Analysis & IOC Extraction

Goal: Analyze a provided JSON log file containing aggregated SIEM-style data and extract all relevant indicators of compromise (IOCs) to support incident response.

Data Source: JSON-based security and system telemetry provided for analysis.

  • Approach:
    • Reviewed JSON log structure to identify relevant fields aligned with incident response requirements.
    • Developed a small Python script to parse and extract relevant indicators of compromise.
    • Focused on producing concise, actionable findings rather than exhaustive raw output.
  • Indicators of Compromise (Summary):
    • Suspicious IP Addresses:
      • 10.14.22.xx — Internal address observed communicating outside the expected subnet range (10.14.10.0/24), indicating potentially abnormal internal segmentation or misuse.
      • 10.14.22.xx — Second internal address with similar out-of-range behavior, suggesting correlation with other anomalous internal activity.
      • 185.217.1.93 — External IP associated with suspicious domain activity and correlated with additional indicators within the dataset.
    • Suspicious Domains:
      • api-svc-metrics.net — Domain name pattern mimicking legitimate telemetry services, potentially used to evade casual inspection.
      • update-check.microsoft-service.com — Domain impersonating a trusted vendor naming convention; inconsistent with legitimate Microsoft update infrastructure.
      • 185.217.1.93 — Domain resolution linked directly to a suspicious external IP address, reinforcing correlation across indicators.
    • Suspicious Files / File Paths:
      • C:\Windows\Temp\mcache.dll — DLL located in a temporary directory rather than a standard system path, suggesting potential misuse or unauthorized execution context.
    • Suspicious Processes or Commands:
      • Parent: services.exe → Process: powershell.exe — PowerShell launched from a service context, which can indicate automated execution.
      • Command: powershell.exe -enc [base64-encoded payload truncated] — Encoded PowerShell command observed; content intentionally truncated in this public writeup.
    • SSL / JA3 Fingerprints:
      • JA3 fingerprint observed on suspicious TLS traffic: 72a589da… (truncated for public reporting).
    • Accounts Potentially Compromised or Misused:
      • NT AUTHORITY\SYSTEM — High-privilege execution context associated with PowerShell activity.
      • _svcAdmin (service account) — Unsuccessful login attempts correlated with suspicious external activity (name retained as provided in dataset).
    • Suspicious Ports or Protocols:
      • Destination port: 4433 — Used across multiple connections; traffic included a JA3 fingerprint observation.
      • Source ports: 52912, 52781, 53100 — Ephemeral source ports associated with the same destination service.

    • Observed high-privilege execution context (NT AUTHORITY\SYSTEM) launching PowerShell with an encoded command, consistent with suspicious automation or scripted execution. A service-style account (_svcAdmin) showed unsuccessful login attempts correlated with suspicious external activity (185.217.1.93). Network indicators included repeated connections to destination port 4433 from multiple ephemeral source ports and TLS fingerprinting signals (JA3 observed). Recommended next steps: isolate affected host(s), review authentication logs for further misuse, collect PowerShell operational logs (if available), and block/monitor the identified external indicators.
